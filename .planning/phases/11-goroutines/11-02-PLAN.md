<task type="auto">
  <name>Task 2: Add parseChannel(), parseSend(), parseReceive()</name>
  <files>src/parser/parser.h, src/parser/parser.cpp</files>
  <action>
    In parser.h private section, add:
    Expr::Ptr parseChannel();
    Expr::Ptr parseSend(Expr::Ptr channel);
    Expr::Ptr parseReceive();
    
    In parser.cpp, implement:
    
    parseChannel() — called from parsePrimary():
    - Match CHANNEL_KEYWORD
    - Try to parse capacity: if next token is NUMBER, consume it as capacity Expr
    - If no capacity (next token is not a number), capacity is null (unbuffered channel)
    - Return ChannelExpr with optional capacity
    
    parseSend() — called from parsePostfix():
    - After parsing a primary/expression, check for ARROW token (->)
    - If found: consume ARROW, parse right-hand side expression as value
    - Return SendExpr with left channel and right value
    - Note: The ARROW token in Harvis Script for send uses `<-` which is LEFT_ANGLE + MINUS
    
    parseReceive() — called from parsePrimary():
    - Check for ARROW followed by LEFT_PAREN or IDENTIFIER or CHANNEL_KEYWORD
    - If pattern is `<-expr`: consume the arrow, parse channel expression
    - Return ReceiveExpr with channel, commaOk=false
    - If pattern is `<-expr, IDENTIFIER`: parse both, return ReceiveExpr with commaOk=true
    
    In parsePrimary(), add:
    if (match(TokenType::CHANNEL_KEYWORD)) return parseChannel();
    if (match(TokenType::LESS) && peek().type == TokenType::MINUS) { advance(); advance(); return parseReceive(); }
    
    In parsePostfix() or parseUnary(), handle `<-expr` as send when preceded by expression.
  </action>
  <verify>cd build && cmake .. && make -j$(nproc) 2>&1 | grep -q "Built target hs"</verify>
  <done>parseChannel, parseSend, parseReceive handle channel creation and send/receive syntax</done>
</task>

<task type="auto">
  <name>Task 3: Add parseSelect() for select statements</name>
  <files>src/parser/parser.h, src/parser/parser.cpp</files>
  <action>
    In parser.h private section, add:
    Stmt::Ptr parseSelect();
    SelectCase parseSelectCase();
    
    In parser.cpp, implement:
    
    parseSelect():
    - Match SELECT_KEYWORD
    - Consume LEFT_BRACE
    - Loop: parse select cases until RIGHT_BRACE
    - Each case starts with CASE_KEYWORD or DEFAULT_KEYWORD
    - For CASE: parse channel expression (or send: channel <- value), then COLON, then parse body statements
    - For DEFAULT: no channel/value, just COLON then body statements
    - Return SelectStmt with cases vector
    
    parseSelectCase():
    - If DEFAULT_KEYWORD: set isDefault=true, consume COLON, parse body statements
    - If CASE_KEYWORD: consume CASE, parse channel expr, parse optional ARROW+value for send, consume COLON, parse body statements
    - Return SelectCase struct
    
    In parseStatement(), add after GO_KEYWORD match:
    if (match(TokenType::SELECT_KEYWORD)) return parseSelect();
  </action>
  <verify>cd build && cmake .. && make -j$(nproc) 2>&1 | grep -q "Built target hs"</verify>
  <done>parseSelect parses `select { case ch <- val: ... case <-ch: ... default: ... }` syntax</done>
</task>
